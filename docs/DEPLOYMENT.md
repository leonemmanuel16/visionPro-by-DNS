# DNS Vision AI — Deployment Guide

## System Requirements

### Minimum (up to 8 cameras)
- CPU: Intel i5 / AMD Ryzen 5 (4 cores)
- RAM: 16 GB
- Storage: 256 GB SSD
- GPU: NVIDIA GTX 1650 (4 GB VRAM)
- Network: Gigabit Ethernet
- OS: Ubuntu Server 24.04 LTS

### Recommended (up to 32 cameras)
- CPU: Intel i7 / AMD Ryzen 7 (8 cores)
- RAM: 32 GB
- Storage: 1 TB NVMe SSD
- GPU: NVIDIA RTX 3060 (12 GB VRAM)
- Network: Gigabit Ethernet
- OS: Ubuntu Server 24.04 LTS

### Enterprise (up to 64 cameras)
- CPU: Intel Xeon / AMD EPYC (16+ cores)
- RAM: 64 GB
- Storage: 2 TB NVMe SSD + NAS for clips
- GPU: NVIDIA RTX 4070 or better
- Network: 10GbE recommended
- OS: Ubuntu Server 24.04 LTS

## Installation

### 1. Prepare the Server

```bash
# Update system
sudo apt update && sudo apt upgrade -y

# Clone repository
git clone https://github.com/leonemmanuel16/dns-vision-ai.git
cd dns-vision-ai

# Run setup script
chmod +x scripts/setup.sh
./scripts/setup.sh
```

### 2. Configure Environment

```bash
nano .env
```

Required settings:
- `DB_PASSWORD` — PostgreSQL password (auto-generated by setup)
- `JWT_SECRET` — JWT signing key (auto-generated by setup)
- `MINIO_PASSWORD` — MinIO password (auto-generated by setup)
- `ONVIF_DEFAULT_USER` — Default camera username
- `ONVIF_DEFAULT_PASS` — Default camera password

### 3. Start Services

```bash
docker compose up -d
```

### 4. Verify

```bash
# Check all services are running
docker compose ps

# Check logs
docker compose logs -f

# Test API
curl http://localhost:8000/health
```

### 5. Access Dashboard

Open a browser and navigate to:
- From server: `http://localhost:3000`
- From network: `http://SERVER_IP:3000`

Login with: `admin` / `admin123`

**Change the default password immediately.**

## Network Configuration

### Camera Network
- Cameras and server must be on the same subnet (or have routing)
- ONVIF discovery uses multicast (239.255.255.250:3702) — ensure multicast is not blocked
- RTSP streams typically use ports 554, 8554

### Firewall Rules
```bash
# Allow dashboard access from LAN
sudo ufw allow 3000/tcp
# Allow API access from LAN
sudo ufw allow 8000/tcp
# Allow go2rtc WebRTC
sudo ufw allow 1984/tcp
sudo ufw allow 8555/udp
```

## Troubleshooting

### Cameras Not Discovered
1. Verify cameras are on the same subnet
2. Check multicast is not blocked by switches/routers
3. Verify ONVIF is enabled on cameras
4. Check credentials in `.env`

### No GPU Acceleration
1. Verify: `nvidia-smi` shows your GPU
2. Verify: `docker run --rm --gpus all nvidia/cuda:12.2.2-base-ubuntu22.04 nvidia-smi`
3. Check NVIDIA Container Toolkit is installed

### High CPU/Memory Usage
1. Reduce `DETECTION_FPS` in `.env` (default: 5)
2. Reduce number of enabled cameras
3. Use sub-streams for detection (lower resolution)
